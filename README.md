# BRDGIT Social Media Automation

Automated social media content workflows built with [n8n](https://n8n.io). These workflows handle organic content generation and topic sourcing for BRDGIT's social media channels.

---

## Workflows

### 1. BRDGIT Organic Content Automation

Runs **Mon / Wed / Fri at 9 AM** to automatically draft social media posts.

**Flow:**
1. Reads available topics from a Google Sheets "Topics" tab
2. Picks a random unused topic (skips topics older than 7 days or already used)
3. Marks the topic as used
4. Loads the BRDGIT Tone & Voice document from Google Drive
5. Uses an AI Agent (Claude via Anthropic API) to generate platform-specific posts
6. Formats drafts and creates them in HubSpot Social (status: awaiting approval)
7. Sends a Slack/email notification when drafts are ready for review

**Credentials required:**
| Credential | n8n Type |
|---|---|
| Google Sheets | `googleSheetsOAuth2Api` |
| Google Drive | `googleDriveOAuth2Api` |
| Anthropic (Claude) | `anthropicApi` |
| OpenAI | `openAiApi` |
| HubSpot | `httpHeaderAuth` |
| Gmail | `gmailOAuth2` |

---

### 2. BRDGIT — Slack Topics to Google Sheets

Runs **every hour** to harvest new content topics from a Slack channel.

**Flow:**
1. Fetches the last 2 hours of messages from the configured Slack channel
2. Filters for messages posted by the **Jarvis** bot
3. Deduplicates against existing topics already in the sheet
4. Uses Claude (Anthropic) + Tavily to research and validate topics
5. Appends new, unique topics to the Google Sheets "Topics" tab

**Credentials required:**
| Credential | n8n Type |
|---|---|
| Google Sheets | `googleSheetsOAuth2Api` |
| Slack | `slackApi` |
| Anthropic (Claude) | `anthropicApi` |
| Tavily | `tavilyApi` |

## Setup

### Prerequisites
- [n8n](https://n8n.io) instance (self-hosted or cloud)
- Google Cloud project with Sheets & Drive APIs enabled
- Anthropic API key
- OpenAI API key
- HubSpot account with Social Media access token
- Slack bot token with `channels:history` scope
- Tavily API key

### Import Workflows

1. In n8n, go to **Workflows → Import**
2. Upload each `.json` file from this repo
3. Configure credentials (see tables above) — in each workflow, open nodes that reference `YOUR_*_CREDENTIAL_ID` and connect your saved credentials
4. Update the following placeholders in the workflow nodes:

| Placeholder | Replace with |
|---|---|
| `YOUR_GOOGLE_SPREADSHEET_ID` | Your Google Sheets spreadsheet ID |
| `YOUR_SLACK_CHANNEL_ID` | The Slack channel ID to monitor |
| `YOUR_WEBHOOK_ID` | Auto-generated by n8n on import |

5. Activate both workflows

### Google Sheets Structure

The Topics spreadsheet needs a sheet named **Topics** with these columns:

| Topic | Last Used | Date Added |
|---|---|---|
| Topic text | (blank until used) | YYYY-MM-DD or blank |

## Notes

- Posts are created as **drafts** in HubSpot and require manual approval before publishing
- The Jarvis Slack bot is expected to post topics to the monitored channel; configure the bot name filter in the "Format & Dedupe Topics" node if your bot has a different name
- Topics older than 7 days are automatically skipped to keep content timely
